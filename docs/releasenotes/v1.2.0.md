# Release Notes — v1.2.0 (Opportunities Terminology & UX Polish)

**Date:** March 12, 2026

**Status:** ✅ COMPLETE

**Audience:** External + internal stakeholders

**Scope:** Client-facing terminology update (Gigs → Opportunities), expanded Opportunity Type filter, and opportunity image specs

**Role legend:**

- Career Builder (role: `client`)
- Talent (role: `talent`)
- Admin (role: `admin`)

---

## Highlights

- **Terminology update**: User-facing labels changed from "Gigs" to "Opportunities" across the platform (navigation, dashboards, forms, error messages, command palette)
- **Opportunity Type filter**: Added "Events" and "Others" as filter options; renamed "Category" to "Opportunity Type" where used for filtering
- **Opportunity image specs**: Recommended dimensions (1200×900 px, 4:3 aspect ratio) documented and surfaced in the cover image uploader so flyers display correctly without distortion
- **Route paths unchanged**: URLs (`/gigs`, `/client/gigs`, etc.) remain the same for stability and bookmarks

---

## By Role Surface

### Career Builder

- **Navigation & labels**: "Gigs" now appears as "Opportunities" in nav, dashboards, and forms
- **Applications page**: "All Gigs" → "All Opportunities"; filter label "Gig" → "Opportunity"
- **Opportunity creation**: "Category" label → "Opportunity Type"; "Events" and "Others" available as types
- **Cover image upload**: Info notice shows recommended 1200×900 px (4:3) for best display

### Talent

- **Browse & apply**: "Gigs" → "Opportunities" in browse filters, detail pages, and apply flows
- **Subscription gate text**: Updated to use "opportunity" terminology
- **Dashboard**: Opportunity-related labels aligned with new terminology

### Admin

- **Opportunity Management**: "Gig Management" → "Opportunity Management"; "Manage all your platform gigs" → "opportunities"
- **Admin gigs list**: Labels updated for consistency

---

## Platform & Trust

- **No breaking changes**: Route paths unchanged; existing links and bookmarks continue to work
- **Constants**: `lib/constants/opportunity-image-specs.ts` — single source of truth for display specs
- **Categories**: `VISIBLE_GIG_CATEGORIES` excludes "other" to avoid confusion with "others" filter option
- **Documentation**: `docs/guides/OPPORTUNITY_IMAGE_SPECS.md` — full spec reference for opportunity cover images

---

## Known Limitations / In Progress

- Career Builder applications currently route to `ADMIN_EMAIL`; per-agent routing may be designed in a future release
- Non-subscribers see generic placeholders instead of full opportunity descriptions (by design; configurable via `getGigDisplayDescription()`)

---

All changes are UI labels, filter options, and documentation. No database migrations or type definition changes required.

## Migration Notes
